            â”‚ # behaviour; however the code below reimplements the default behaviour
            â”‚ # as a starting point for convenient modification.

            â”‚ # branch is going to be updated:
            â”‚ commit=$1

            â”‚ # so, it must not modify the index or the working tree).
            â”‚ die () {
            â”‚ 	echo >&2 "$*"
            â”‚ 	exit 1
            â”‚ }

            â”‚ # index to bring them to the desired state when the tip of the current
            â”‚ # branch is updated to the new commit, and exit with a zero status.
            â”‚ #
            â”‚ # For example, the hook can simply run git read-tree -u -m HEAD "$1"
            â”‚ # in order to emulate git fetch that is run in the reverse direction
            â”‚ # with git push, as the two-tree form of git read-tree -u -m is
            â”‚ # essentially the same as git switch or git checkout that switches

            â”‚ # not interfere with the difference between the branches.

            â”‚ # the push_to_deploy() function in builtin/receive-pack.c.
            â”‚ #

            â”‚ # not from the working tree, so if you want to perform operations on
            â”‚ # the working tree, you will have to adapt your code accordingly, e.g.

            â”‚ 	die "Working directory has unstaged changes"
